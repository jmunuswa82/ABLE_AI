Treat this as a P0 architecture and implementation correction for the Ableton ALS pipeline.

1. Deploy/Export gets stuck in a perpetual “Checking” state

==================================================

- pipeline cards show all stages green
- export card/button remains in “Checking” forever
- export CTA does not transition to ready/download

This indicates broken export state resolution.

Find and fix all possible causes:
- export readiness polling wrong endpoint
- frontend polling different project_id than current active project

- race condition between “strategy generated” and “artifact compiled”
- polling loop does not stop on terminal states

Required fix:
- introduce explicit export lifecycle states:
  NOT_AVAILABLE
  AWAITING_SELECTION
  READY_TO_COMPILE
  COMPILING
  READY
  FAILED

- no generic “Checking” forever
- every polling flow must:
  1. start with known state
  2. move deterministically
  3. stop on READY or FAILED
  4. expose failure cause if timeout or missing artifact

Add timeout protection:
- if state is COMPILING too long, surface diagnostics
- if file artifact missing after backend claims READY, show export artifact mismatch error
- never spin forever silently

PART 4 — MUTATION MODEL MUST BE TYPED, NOT FREEFORM
==================================================

Do NOT let AI text directly mutate XML.

All selected improvements must be converted into typed mutation objects.

Implement a canonical mutation schema like:

{
  "mutationId": "mut_001",
  "type": "ADD_AUTOMATION_ENVELOPE",
  "category": "automation",
  "enabled": true,
  "target": {
    "trackId": "...",
    "deviceId": "...",
    "parameterId": "...",
    "clipId": "...",
    "bars": { "start": 97, "end": 105 }
  },
  "payload": {
    "points": [
      { "bar": 97, "value": 0.15 },
      { "bar": 101, "value": 0.62 },
      { "bar": 105, "value": 0.25 }
    ]
  },
  "constraints": {
    "requiresExistingTrack": true,
    "requiresValidParameterRef": true
  },
  "sourceEvidence": {
    "reason": "No transition automation found before breakdown",
    "confidence": 0.92
  }
}

Other mutation types:
- ADD_LOCATOR
- ADD_MIDI_CLIP
- EXTEND_ARRANGEMENT
- ADD_TRACK
- RENAME_TRACK
- ADD_RETURN_SEND_AUTOMATION
- ADD_UTILITY_GAIN_AUTOMATION
- ADD_FILTER_SWEEP_AUTOMATION
- INSERT_TRANSITION_GAP
- ADD_OUTRO_SECTION
- ADD_SIDECHAIN_ROUTING_METADATA (only if structurally valid)
- ADD_PLACEHOLDER_AUDIO_CLIP_REFERENCE (if supported safely)

Important:
Only safe, structurally supported mutations may be directly compiled into `.als`.
Unsafe or ambiguous suggestions must be marked:
- manual_only = true
- not directly exportable
