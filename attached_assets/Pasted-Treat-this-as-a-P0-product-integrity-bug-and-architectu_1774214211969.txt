- the app appears to have disconnected frontend screens instead of one unified project pipeline

This is unacceptable.

Upload `.als` -> persist project -> analyze actual file -> generate real strategy from that file -> enable export based on same project state

No placeholder analysis.
No disconnected screen logic.

==================================================

3. Backend stores artifact and creates project record
4. Backend returns:
   - project_id
   - artifact_id
   - original filename
   - file size
   - upload status
5. Frontend stores active project_id in central state/router
6. “Initiate Pipeline” now acts on that existing project_id

Critical rule:
After successful upload, the file picker must never reopen automatically from “Initiate Pipeline”.

Add separate buttons:
- Upload ALS
- Replace File
- Initiate Analysis Pipeline

==================================================
BACKEND API CONTRACT
==================================================

Implement or refactor backend endpoints into a clean stateful flow:

POST   /api/projects/upload
  -> accepts `.als`
  -> creates project + stores file
  -> returns project metadata

GET    /api/projects/:projectId
  -> returns full project lifecycle state

POST   /api/projects/:projectId/initiate-pipeline
  -> starts ingest/parse/analyze/plan jobs for the already uploaded file
  -> never requires reopening file picker

GET    /api/projects/:projectId/status
  -> returns unified state:
     uploadStatus
     ingestStatus
     parseStatus
     analysisStatus
     planStatus
     exportStatus
     progress percentages
     latest error

GET    /api/projects/:projectId/analysis
  -> returns actual parsed/analyzed metrics for that project only

GET    /api/projects/:projectId/strategy
  -> returns actual completion strategy derived from project analysis only

GET    /api/projects/:projectId/export
  -> returns export readiness and artifact info

POST   /api/projects/:projectId/export-als
  -> exports modified `.als` only after successful analysis/mutation/validation

Important:
Strategy and Export must not read from any global demo JSON or static file.
Everything must be keyed by project_id.
